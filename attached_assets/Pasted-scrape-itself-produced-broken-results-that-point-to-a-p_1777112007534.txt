**Bug 1 (HIGH): Mode defaults to "Online" when extractor can't determine mode**

All 44 KBS courses staged with `Mode: Online`. KBS courses are NOT online — they're on-campus across 6 cities (Adelaide/Brisbane/Gold Coast/Melbourne/Perth/Sydney) per the page content. The scraper has correctly extracted these locations as the `Course Location` field, then for some reason set `Mode: Online`.

My guess: when the mode extractor can't find an explicit "On Campus" / "Online" / "Blended" string on the page, it defaults to `Online`. This is the wrong default. Two implications:

1. KBS courses are misclassified — they're on-campus, not online
2. **The Torrens online-only filter (when it ships) will incorrectly REJECT these KBS courses** because they're labeled Online when they're actually on-campus

Proposed fix: when mode can't be determined explicitly, derive it from `course_location`:
- If location contains 1+ city names (Adelaide, Brisbane, Sydney, Melbourne, Perth, etc.) → `On Campus`
- If page mentions "online" in body but no city locations → `Online`
- If both → `Blended`
- If neither → `Unknown` (NOT `Online`)

Note: if you ship the Torrens filter ticket BEFORE fixing this, KBS will look like 44 wrongly-rejected courses. Fix this first, OR ship them together.

**Bug 2 (HIGH): No follow-link extraction for centralized fees / entry requirements**

KBS course pages link to centralized admissions pages:
- `https://www.kbs.edu.au/admissions/fees/international-fees`
- `https://www.kbs.edu.au/admissions/fees/domestic-fees`
- `https://www.kbs.edu.au/admissions/entry-requirements/international-entry-requirements`
- `https://www.kbs.edu.au/admissions/entry-requirements/english-entry-requirements`

These pages presumably contain a fee table indexed by course code (CRICOS code is shown on the course page — 110274C for the Master of IT) and entry requirements that apply uni-wide (English requirements for international students are often the same across all programs).

Current scraper architecture: "course page wins" with PDF / image cache as fallback. No mechanism for following central admissions links.

Proposed feature: **central admissions cache per uni**. When discovery runs, identify candidate central pages (URL patterns: `/admissions/fees`, `/admissions/entry-requirements`, `/fees`, `/english-requirements`, etc.), fetch each one ONCE per scrape, parse into a cache keyed by `(course_code, level, intl/dom)`. Per-course extraction then consults the cache when on-page extraction fails for fees / IELTS.

This is similar in spirit to the sibling cache for English requirements, but cross-cutting all courses in a uni. Worth designing carefully — KBS is the first instance but USyd / UNSW / ANU will hit the same pattern at much larger scale.

**Bug 3 (MEDIUM): Duration extractor truncates compound durations**

KBS page literally says: "Typical Duration / Standard Study Option **1** year, **8** months"

Scraper extracted: `1 Year`. Lost the "8 months" part. Real value is 1.67 years or 20 months.

This is a duration.py extractor bug — only catches first numeric+unit pair, doesn't compose. Fix: when input matches `<N> year(s)?,? <M> month(s)?` pattern, output `N years M months` or normalize to total months / total years with decimal.

Note: this also overlaps with Bug D from Torrens (float precision `1.7000000476837158 Year`). When you fix that one, account for compound durations too.
