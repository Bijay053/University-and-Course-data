**4. Two instances of the celery zombie-job problem during the run.**

This is a separate prod infrastructure bug — the API enqueues a job to the DB but the celery dispatch to Redis is failing silently in some race condition (probably immediately after a `systemctl restart` while celery is still warming up).

Worth a separate bug ticket: "API returns `queued` for scrape jobs but celery never picks them up if dispatched within ~10s of worker restart."

Workaround for now: always `sleep 15` between `systemctl restart` and `curl -X POST scrape/start`. Better fix: make the API verify the task actually landed in Redis before returning 200.
