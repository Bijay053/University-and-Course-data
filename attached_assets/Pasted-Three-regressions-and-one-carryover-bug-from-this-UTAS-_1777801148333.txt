Bug 1 — fee_term defaults to "Full Course" when set via AI fallback path
Pattern visible across the dashboard:
CourseSource pathDashboard shows93A, 93J, 93LGemini primary set fee_term=Annual directly"Annual" ✅L6bGemini primary set fee_term=Annual directly"Annual" ✅H3x, S4a, A3a, M3x, L3c, M3s, P3i, P3l, P3r, P3t, A3m, 13O, 13R, P3e, P7c, 53F, P3o, E7g, B3a, B3b, P3j, 43B, M3s, H7lregex got the fee, AI fallback fired for currency/duration"Full Course" ❌
Every Gemini raw response in the log includes "fee_term": "Annual" for the courses with fees. But the dashboard shows "Full Course" for any row where the fee was caught by regex first (regex is hitting Tuition fee based on a rate of $X AUD per standard, full-time year of study and extracting just the number, not the period qualifier).
Root cause: when regex sets international_fee directly and Gemini's response is then merged for AI fallback fields, fee_term is either not being merged, OR the row's default fee_term="Full Course" is winning over Gemini's Annual.
Fix: In the row merge logic, when Gemini emits fee_term, that value should overwrite the default. AI fallback is not the primary issue — the primary issue is that regex extracts the fee number without setting fee_term, leaving it at default "Full Course".
