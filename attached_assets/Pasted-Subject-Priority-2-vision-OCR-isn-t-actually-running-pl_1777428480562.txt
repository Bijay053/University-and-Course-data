This reveals two separate problems:
Problem 1: Vision OCR never fires for any course in the scrape
Looking at the full log, there are zero [VISION] / [per_course_vision] log entries for any of the 9 ASAHE courses. The pipeline jumps from browser timeout straight to AI fallback to PG-SKIP wipe — vision OCR is skipped entirely.
Most likely cause: this line at the end of the log:
[GEMINI] No Gemini PRIMARY calls billed this scrape (key unavailable or budget exhausted)
Vision OCR uses Gemini Vision, which shares quota with Gemini text. If quota is exhausted, vision OCR silently fails too. This is the "Gemini API quota exhaustion" pain point from §10 of the architecture doc.
Please verify:

Is the Gemini API key set in production? echo $GEMINI_API_KEY | head -c 8 should show non-empty.
If the key is set, is the quota exhausted? Check the Google AI Studio console for the project's daily token usage.
Add an explicit log line at the start of vision OCR: [VISION] attempting OCR on {image_url} (and [VISION FAIL] {image_url}: {reason} on failure) so we can see whether vision is being skipped because of quota, missing image candidates, or never being called.

Until vision OCR demonstrably fires, Priority 2 isn't shippable.
