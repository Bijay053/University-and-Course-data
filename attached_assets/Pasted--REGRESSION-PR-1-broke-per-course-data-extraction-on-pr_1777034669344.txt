Two distinct problems:

Browser + vision fallbacks return empty data on URLs that clearly contain it (these pages load fine in a real browser).
The one successful /mba extraction's values appear to be cross-contaminating other rows — those IELTS/PTE/TOEFL/CAE values in the MBA - Information Systems row look like they came from /mba, not from /mba/mba-information-systems (which logged empty).
The 60s AI timeout is firing — fallback bails before completing on some URLs.

backend-py/app/services/scraper/per_course_browser.py — extractor returning empty when page has data
