## Two suspicious patterns

**1. `out=10-15` tokens consistently.** Across 30+ calls, the model output is always 10-15 tokens. That's just enough to return `{}` or `{"international_fee": null, "ielts_overall": null, ...}`. The model is responding *consistently* with empty/null structure — not a transient failure or rate limit.

**2. `in=~2780` tokens consistently.** Input size is uniform across all KBS courses, which suggests `html_to_text` is producing roughly the same processed text for every course. KBS course pages render different content per program (fees, durations, IELTS), but Gemini is seeing roughly the same input size every time.

This points to one of two things:
- `html_to_text` is stripping the actual course content and feeding Gemini boilerplate (header/footer/nav repeated across all pages)
- The prompt instructs the model to extract fields that don't match what's actually present in the input
