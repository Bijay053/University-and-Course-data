STEP 5 — EXTRACTION LOGIC TO IMPROVE IN scrape.ts
==================================================

Implement or fix these parsers:

1. Currency logic
If explicit currency exists, use it.
Otherwise use country fallback:
- Australia AUD
- New Zealand NZD
- Canada CAD
- USA USD
- UK GBP

2. Fee logic
Always prefer international fee.
Never save domestic fee as the main fee.
If both exist, classify by nearby labels and select international.
Extract:
- internationalFee
- feeTerm
- feeYear
- currency

3. Fee term normalization
Normalize:
- per semester = Semester
- per trimester = Trimester
- per term = Term
- annual, annually, per year, yearly = Annual
- full course, total course fee = Full Course

4. Fee year logic
Extract year only if directly tied to fee text such as 2026 fee or fees for 2026 academic year.

5. Study mode normalization
Normalize:
- on campus = On Campus
- online = Online
- blended = Blended
- hybrid = Blended
- online and on campus = Blended

6. IELTS logic
If text says overall 6.0 with no band below 5.5 then save:
- overall 6.0
- listening 5.5
- reading 5.5
- writing 5.5
- speaking 5.5

Apply same pattern to PTE and TOEFL when wording gives overall plus minimum component score.

7. Duration and duration term
Split numeric duration from unit:
- 2 years
- 18 months
- 6 semesters
- 4 trimesters

8. Intake parsing
Split intake month and intake day where exact dates exist.
If only month exists, save month and leave day blank.

9. Academic requirement by country
If a course or shared page shows different country rows, preserve:
- academicCountry
- academicLevel
- academicScore
- scoreType
- otherRequirement

If this parsing is complex, only then use AI fallback.
