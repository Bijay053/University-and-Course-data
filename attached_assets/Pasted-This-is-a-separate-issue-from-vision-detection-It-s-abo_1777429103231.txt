The whole point of scraped_field_evidence is that every value on a course must be backed by a provable source.The architecture doc explicitly states (Section 8):
enforce_source_evidence: Drops any field with no snippet + source URL proof. Prevents ghost values with no proof.

WITH target AS (
  SELECT id, course_name, scrape_job_id,
         ielts_overall, pte_overall, toefl_overall,
         created_at, status
  FROM scraped_courses
  WHERE course_name ILIKE '%Master of Information Technology (Artificial Intelligence)%'
  ORDER BY created_at DESC
)
SELECT 'COURSE_ROW' AS type, t.id::text AS row_id,
       t.scrape_job_id AS job, t.status,
       'ielts=' || COALESCE(t.ielts_overall::text,'NULL') ||
       ' pte=' || COALESCE(t.pte_overall::text,'NULL') ||
       ' toefl=' || COALESCE(t.toefl_overall::text,'NULL') AS values,
       t.created_at::text AS info
FROM target t
UNION ALL
SELECT 'EVIDENCE' AS type, sfe.id::text,
       sfe.method,
       sfe.field_key,
       sfe.normalized_value,
       sfe.selected::text || ' | ' || COALESCE(sfe.source_url, '(no url)')
FROM scraped_field_evidence sfe
WHERE sfe.scraped_course_id IN (SELECT id FROM target)
  AND sfe.field_key IN ('ielts_overall','pte_overall','toefl_overall')
ORDER BY type, info DESC;This shows:

How many scraped_courses rows exist for this course (Cause C)
What values are stored in their columns
All evidence rows for the English fields, with which are selected and their source_url
