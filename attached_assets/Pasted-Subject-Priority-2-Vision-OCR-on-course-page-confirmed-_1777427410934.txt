Subject: Priority 2 — Vision OCR on course page (confirmed bug: wrong ASAHE English data)
Hi,
Manual inspection of https://www.asahe.edu.au/courses/master-of-information-technology-software-application-development confirmed that English requirements are stored as a PNG on the course page, and the values in that image do NOT match what's in the central admissions PDF.
Image URL: https://cdn.prod.website-files.com/68660d9386e56f070b7bec59/696d888d52876a0edc8b3450_MaSTER.png
Current scraper behavior: vision OCR is configured as a fallback for when text extraction fails. But for ASAHE the text extraction technically "succeeds" because the central PDF parser fills the slots — so vision OCR never runs on the course page. Result: every ASAHE Masters course currently shows the wrong IELTS/PTE/TOEFL/CAE/DET in production.
Fix needed:
Move course-page vision OCR from "fallback when nulls remain" to "always runs in Phase A, before any university-wide source." This is the change we discussed for Priority 2.

In single_course.py, after the existing course-page extractors (regex/Gemini/browser) but before Phase B (PDF backfill, central page backfill):

Image-candidate filtering
Don't OCR every image on every page — that exhausts the Gemini quota fast. Filter to candidates first:
pythondef _course_page_has_candidate_images(html: str) -> list[str]:
    """Return URLs of images likely to contain English requirement tables."""
    soup = BeautifulSoup(html, "lxml")
    candidates = []
    for img in soup.find_all("img"):
        src = img.get("src") or ""
        alt = (img.get("alt") or "").lower()
        # filename hints (lowercased filename for matching)
        filename = src.lower().split("/")[-1]

        keywords_in_filename = any(k in filename for k in [
            "ielts", "english", "language", "requirement", "entry",
            "admission", "master", "bachelor"
        ])
        keywords_in_alt = any(k in alt for k in [
            "english", "ielts", "requirement", "entry", "admission"
        ])
        # ASAHE-style: filename like "MaSTER.png", "BACHELOR.png"
        looks_like_level_image = any(
            level in filename for level in ["master", "bachelor", "diploma", "phd"]
        )

        if keywords_in_filename or keywords_in_alt or looks_like_level_image:
            candidates.append(src)
    return candidates

Level-aware image selection
ASAHE has separate images: MaSTER.png for Masters, presumably BACHELOR.png (or similar) for Bachelors. Pick the right image based on the course's degree_level:
pythondef _select_image_for_course(candidate_urls: list[str], degree_level: str) -> str | None:
    level_lower = (degree_level or "").lower()
    for url in candidate_urls:
        filename = url.lower().split("/")[-1]
        if "master" in level_lower and "master" in filename:
            return url
        if "bachelor" in level_lower and ("bachelor" in filename or "undergrad" in filename):
            return url
    # fallback: first candidate
    return candidate_urls[0] if candidate_urls else None
