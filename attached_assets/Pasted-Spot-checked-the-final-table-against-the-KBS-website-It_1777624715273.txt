**What actually fixed it (commit 1a1c95a):**

Two separate bugs were stacked on top of each other:

**Fix 1 — Diploma column key.** The column-keyed parser was emitting all rows under `undergraduate` regardless of source column. So when Bachelor+PG (6.0) and Diploma (5.5) were both parsed, the Bachelor value overwrote the Diploma value. Renaming the Diploma column key to `diploma` lets Diploma courses pick up their own 5.5/46 values without contention.

**Fix 2 — Override guard for low-authority sources.** Even after the parser fired correctly and cached the right values, individual course rows were still showing 6.5 from `ai_fallback`. The orchestrator was treating Gemini's hallucinated 6.5 as already-set, so the central-page values from the verified-URL parser couldn't replace them. Now `ai_fallback` and `gemini_primary` are explicitly low-authority — if a verified central-page URL has a better value, it overrides.

The Path 1 + Path 2 split in `single_course.py` is the right architecture for this: a clear authority hierarchy with central-page values from verified URLs winning over AI guesses.
