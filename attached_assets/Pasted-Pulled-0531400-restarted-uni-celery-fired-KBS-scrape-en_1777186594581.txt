## Possible causes

1. **Log level mismatch** — the GP-DEBUG lines are using `logger.debug()` but the worker is configured at INFO level, so DEBUG messages get filtered out. The `[GEMINI]` lines (which DO appear) are using a different log level (`info` or `print`).

3. **Wrong file location** — the GP-DEBUG calls were placed in a code path that isn't reached for KBS courses.
4. **Print vs log** — if the lines use `print()` instead of the logger, they may be going to stdout which Celery captures differently.
