**Tomorrow's housekeeping sequence looks right:**

The four-step opening you outlined is the correct order. A few minor refinements:

**Step 1 — git working-tree investigation:**

```
git log --oneline --diff-filter=D --all -- backend-py/scripts/bulk_approve.py
```

This specifically shows commits that DELETED bulk_approve.py from the tree. If it returns nothing, the file was never committed-deleted — which means the disappearance was a working-tree event (uncommitted), not a git history event. In that case `git reflog | head -20` is the next step — it'll show stash/reset/checkout that affected the working tree without changing history.

**Step 2 — git clean preview:**

```
git clean -nfd
```

The `-n` (dry-run) is exactly right — don't actually clean until we've reviewed the list. If the list contains anything that looks like work-in-progress (e.g. a `.env.local`, an experimental script not yet committed), pause before running without `-n`.

**Step 3 — pg_dump backup:**

Good. Worth adding `gzip` so the backup compresses (university_portal at 623 courses is small but still benefits):

```
mkdir -p /root/backups
pg_dump university_portal | gzip > /root/backups/uni_portal_$(date +%Y%m%d_%H%M%S).sql.gz
```

**Step 4 — ACU diagnostic:**

Good. Same shape as the Bond/Flinders diagnostics that worked. Run it before deciding whether ACU needs an enrichment pass first or can go straight to bulk_approve.

One addition worth pre-running: a Bond IELTS=6.5 ground-truth spot-check, since we deferred it from today:

```
sudo -u postgres psql university_portal -c "
SELECT id, name, ielts_overall,
       (SELECT extraction_method->>'ielts_overall'
        FROM scraped_courses sc WHERE sc.id = (
          SELECT MAX(id) FROM scraped_courses
          WHERE university_id=22 AND course_name=courses.name
        )) AS source
FROM courses
WHERE university_id=22 AND ielts_overall IS NOT NULL
ORDER BY random() LIMIT 5;"
```

Then open the 5 random Bond URLs and verify against bond.edu.au. ~3 minutes. If sources are all `central_page:*`, Bond is verified clean. If `ai_fallback` shows up multiple times, we have the same Gemini hallucination pattern KBS had — catchable before it spreads to admin approval.
