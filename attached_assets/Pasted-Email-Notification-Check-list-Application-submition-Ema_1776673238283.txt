Fix notification click redirection
Check all notifications and make sure each one is clickable and redirects to the exact page and exact tab related to that notification, not just to a generic page.
Example:
 If the notification is for an agreement signature, clicking it should open the Settings page and automatically show the Agreement tab, not just redirect to the main Settings page.
Requirement:
 Each notification type must redirect users to its correct destination page, tab, section, or record based on the notification content.
Expected behavior:
Notification must be clickable
Redirect must go to the correct module
Correct tab should open automatically
If needed, the related record or detail view should also open
Do not send users to a generic landing page when a specific destination exists
Examples:
Agreement sign notification → Settings > Agreement tab
Payment release request notification → Payment section with related request
Task notification → Related task detail or correct task tab
Application notification → Related application page with correct section open
Event notification → Related event detail page
Testing:
 Review all notification types one by one and confirm:
click works
route is correct
correct tab opens
related data is shown properly
