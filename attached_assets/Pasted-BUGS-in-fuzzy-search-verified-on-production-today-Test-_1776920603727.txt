BUGS in fuzzy search — verified on production today:
Test results (commit a1f8c07):

?location=sidney → 19 results ✓
?location=aussie → 0 results ❌
?q=bachlor%20nursing → 21 results ✓ + "did you mean" suggestion
?q=bachlor%20nursing&location=sidney → 0 results ❌ (combining breaks it)

Root causes:
A. Location aliases not supported. "aussie", "oz", "syd", "melb" etc. are empty returns. Add alias expansion.
B. Combined q + location uses AND with strict location matching even though the course name uses fuzzy. Probably the location WHERE clause is ILIKE %location% while course uses % operator.
C. "Did you mean" is returned but not clickable in the UI. User sees "Did you mean: Bachelor Nursing" as plain text with no way to click it to re-run the search.
D. Zero-result fallback missing. When combined filters = 0, frontend should try each filter separately and suggest "we have 21 courses matching 'nursing' and 19 in Sydney — remove a filter?"

Fixes:

sql-- When location returns 0, find closest match in cities/countries
SELECT value, levenshtein(LOWER(value), LOWER($location)) as dist
FROM (
  SELECT DISTINCT city as value FROM universities WHERE city IS NOT NULL
  UNION
  SELECT DISTINCT country FROM universities WHERE country IS NOT NULL
  UNION
  SELECT DISTINCT unnest(string_to_array(course_location, ',')) FROM courses WHERE course_location IS NOT NULL
) candidates
