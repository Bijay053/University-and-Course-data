Production impact: 148 out of 184 CSU courses had names like "Bachelor Business Studies" instead of "Bachelor of Business Studies". Actual page title at https://study.csu.edu.au/international/courses/bachelor-business-studies is "Bachelor of Business Studies".

Scraper fix needed in course-name extraction/normalization:

The cleaning pipeline strips "of", "in", "and" etc. as if they were stopwords. Stop doing that for course names — keep all words from the original page title.
Only strip leading/trailing noise:

University name prefix ("Charles Sturt — ")
"Overview" / "Course details" suffix
Pagination markers, breadcrumb residue


Add URL-slug validation: If the extracted name is "Bachelor X" but the URL slug contains "bachelor-of-X", prefer the slug-derived name. Example:
typescriptfunction validateNameAgainstSlug(extractedName: string, url: string): string {
  const slug = url.split('/').pop() ?? '';
  const slugWords = slug.toLowerCase().split('-');
  const nameWords = extractedName.toLowerCase().split(/\s+/);
  
  // If slug has "of"/"in"/"and" but extracted name doesn't, reconstruct from slug
  const prepositions = ['of', 'in', 'and'];
  const slugHasPrep = slugWords.some(w => prepositions.includes(w));
  const nameHasPrep = nameWords.some(w => prepositions.includes(w));
  
  if (slugHasPrep && !nameHasPrep) {
    // Reconstruct from slug with Title Case
    return slug.split('-').map(w => 
      prepositions.includes(w) ? w : w.charAt(0).toUpperCase() + w.slice(1)
    ).join(' ');
  }
  return extractedName;
}

Write a test case that covers "Bachelor Business Studies" (extracted) with URL bachelor-business-studies should stay as-is, but URL bachelor-of-business-studies should become "Bachelor of Business Studies".

typescriptconst VALID_AU_CAMPUSES = new Set([
  // Major AU cities
  'sydney', 'melbourne', 'brisbane', 'perth', 'adelaide', 
  'hobart', 'canberra', 'darwin', 'gold coast', 'newcastle',
  // CSU campuses
  'bathurst', 'albury-wodonga', 'albury', 'wodonga', 'dubbo', 
  'port macquarie', 'wagga wagga', 'orange', 'canberra',
  // USQ campuses
  'toowoomba', 'springfield', 'ipswich',
  // Other common
  'holmesglen', 'external', 'online', 'uni wide'
]);

const LOCATION_BLOCKLIST_PATTERNS = [
  /click here/i,
  /google/i,
  /feedback/i,
  /trouble accessing/i,
  /session \d/i,
  /^test$/i,
  /^on campus$/i,
  /^external$/i,
  /\bjilin\b|\btianjin\b|\byangzhou\b|\byunnan\b/i,  // exchange partner artefacts
];

function validateCourseLocation(raw: string | null): string | null {
  if (!raw) return null;
  if (raw.length > 150) return null;  // too long = prose, not location
  
  // Reject blocklist patterns
  for (const pattern of LOCATION_BLOCKLIST_PATTERNS) {
    if (pattern.test(raw)) return null;
  }
  
  // Split on comma, validate each piece against known campus list
  const parts = raw.split(',').map(p => p.trim().toLowerCase());
  const validParts = parts.filter(p => 
    VALID_AU_CAMPUSES.has(p) || 
    // Allow if it's 2-30 chars and no weird chars
    (p.length >= 2 && p.length <= 30 && /^[a-z\s-]+$/.test(p))
  );
  
  if (validParts.length === 0) return null;
  
  // Return original casing from raw, filtered to valid parts
  return raw.split(',')
    .map(p => p.trim())
    .filter((_, i) => validParts.includes(parts[i]))
    .join(', ') || null;
}
