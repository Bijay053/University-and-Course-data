**Question 2 — The Bond enrichment data has a 6.5 monoculture worth one spot-check.**

Looking at the bond_enrich log, a striking pattern: nearly every course has `ielts=6.5`. Out of 117 enriched rows:
- ~10 show ielts=7.0 (Doctor of Physiotherapy, Master of OT, Master of Architecture, etc.) — these track with healthcare/professional programs
- ~6 show ielts=6.0 — some diplomas
- The rest (~100) show ielts=6.5

Is this real, or is it the same Gemini-hallucinates-6.5 pattern we caught on KBS? Bond.edu.au actually does set most undergrad programs at IELTS 6.5, so 6.5 monoculture isn't necessarily wrong — but worth one ground-truth check:

```
# Pick three courses at random from different levels and check
sudo -u postgres psql university_portal -c "
SELECT id, course_name, ielts_overall,
       extraction_method->>'ielts_overall' AS source,
       course_website
FROM scraped_courses
WHERE university_id = 22 AND ielts_overall IS NOT NULL
ORDER BY random()
LIMIT 5;"
```

Then open the `course_website` URL for each and verify the IELTS value against the actual Bond page. If `source = 'ai_fallback'` on more than 1-2 of them, we have a Gemini-substitution problem like KBS had. If `source = 'central_page:...'` or similar, it's real.

The specific concern is that the 9 "WARNING ... no data-program-detail-url found" and "HTML fetch failed" rows mostly ended up with `ielts=None` (correct — no data, no value), but the 117 successful ones being so heavily 6.5 is statistically suspicious. One spot check resolves it.
