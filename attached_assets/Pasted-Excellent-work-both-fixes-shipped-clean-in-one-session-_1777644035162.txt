Two refinements to the audit plan before tomorrow:

**Audit query needs to actually detect cross-contamination, not just count vision_cached usage.** The query you wrote tells us how many rows used `per_course_vision_cached` for IELTS, but not how many of those have evidence pointing to an image from a different course. The cross-contamination count is what tells us whether we need a remediation plan or just monitoring going forward.

Something like (column names probably wrong, you'll know the actual schema):

```sql
SELECT 
  COUNT(*) FILTER (WHERE extraction_method->>'ielts_overall' = 'per_course_vision_cached') AS vision_cached_total,
  COUNT(*) FILTER (
    WHERE extraction_method->>'ielts_overall' = 'per_course_vision_cached'
    AND evidence->>'ielts_overall' NOT LIKE '%' || regexp_replace(source_url, '.*/([^/]+)$', '\1') || '%'
  ) AS vision_cached_cross_contaminated
FROM scraped_courses
WHERE status IN ('pending','approved','promoted');
```
