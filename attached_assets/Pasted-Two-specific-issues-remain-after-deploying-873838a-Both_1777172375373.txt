## Issue 1: "MBA – Two Specialisations" still passes filter

The `_GENERIC_CATEGORY_NAMES` addition for `"two specialisations"` doesn't match because the staged course_name is `MBA – Two Specialisations` (with prefix from the title-tag extractor).

Log shows the course flowing through:
```
[STAGE] saved: MBA – Two Specialisations
```

The set lookup `name.lower() in _GENERIC_CATEGORY_NAMES` requires exact match. `"mba – two specialisations"` != `"two specialisations"`.

**Better fix — reject by URL pattern.** The URL `/mba-master-of-business-administration/two-specialisations` is a stable category-page indicator. Add to URL-based reject rules:
```
if url_path.endswith('/two-specialisations'):
    reject('category_landing_page')
```

Cleaner than name-matching since the title extractor will keep adding prefixes that break exact matches.

## What I want

The PG-skip rule needs to be per-university configurable, not a blanket global. Three options:

**Option A (preferred): per-uni opt-in flag.** Add a config `central_english_pg_skip: true/false` per university. Default false. Set to true ONLY for ASA (and any future uni where the central page demonstrably hides PG scores). KBS = false.

**Option B: detect at parse time.** Check whether the central English page contains level-specific markers (e.g., "Postgraduate", "Master's", "Bachelor's" headers). If only one set of values renders without level distinction, treat it as level-agnostic and apply to all. If multiple levels are detected and only one was extracted, apply the skip.

**Option C: mark the central data with a `central_english_level` field.** When the parser extracts central English data, label whether it's level-specific (`undergraduate`/`postgraduate`/`all_levels`). The fallback block respects the label.

C is cleanest architecturally. A is fastest to ship. Either is fine.
