OPTIMIZATION: Scrape wastes 5 minutes stuck on "Trying vision-AI on 6 candidate image(s) for requirements table..." on every VIT scrape.
The scrape eventually works and produces correct data, but this one step burns 5 minutes because it's sequential and has no timeout. Meanwhile the per-course modal extraction (which actually works) is blocked from starting until it finishes.

FIX 1 — Make requirements-page vision NON-BLOCKING (run in background)
The per-course loop should start IMMEDIATELY without waiting for the requirements-page vision scan. Requirements-page data is a fallback anyway — the per-course modal scraping gets the real data per course.
Find the block in scrape.ts that logs "Trying vision-AI on 6 candidate image(s) for requirements table..." and wrap it in a fire-and-forget promise:
typescript// OLD: awaits the vision scan, blocks everything
const reqsData = await extractRequirementsViaVision(candidateImages);

// NEW: start it in background, continue with per-course extraction
const reqsPromise = extractRequirementsViaVision(candidateImages)
  .catch(e => {
    addLog(job, 'info', { message: `[requirements-vision] background scan failed: ${e.message}` });
    return null;
  });

// Continue immediately — don't await
// Per-course loop starts NOW, not 5 minutes later
Later, when per-course extraction needs the requirements data as fallback, check if reqsPromise has resolved yet:
typescript// When a course has no English data and needs fallback
const reqsData = await Promise.race([
  reqsPromise,
  new Promise(r => setTimeout(() => r(null), 100)) // don't wait more than 100ms
]);
If the background scan finished by then, great. If not, use sibling cache or null — per-course modal usually has what we need anyway.

FIX 2 — Hard timeout on the requirements-page vision loop
Even in background, cap total time at 30 seconds (not 5 minutes):
typescriptconst VISION_BUDGET_MS = 30_000;
const startTime = Date.now();

for (let i = 0; i < candidateImages.length; i++) {
  const elapsed = Date.now() - startTime;
  if (elapsed > VISION_BUDGET_MS) {
    addLog(job, 'info', { 
      message: `[requirements-vision] Stopped at image ${i}/${candidateImages.length} (budget: ${Math.round(elapsed/1000)}s)` 
    });
    break;
  }
  
  try {
    const result = await Promise.race([
      callGeminiVision(candidateImages[i]),
      new Promise((_, reject) => setTimeout(() => reject(new Error('image timeout')), 10_000))
    ]);
    if (hasValidEnglishData(result)) break; // found it, stop early
  } catch (e) {
    addLog(job, 'info', { message: `[requirements-vision] img ${i+1} skip: ${e.message}` });
    continue;
  }
}
10-second per-image cap × 3 images = 30s max. Currently each image takes ~50s × 6 images = 5 min.

FIX 4 — Add stuck-step detection
If any single step takes > 90 seconds, log a warning so we can see where time is spent:
typescriptconst stepStart = Date.now();

  });

  await doTheStep();

  clearTimeout(warnTimer);
  addLog(job, 'info', {

  });
}
Apply to these steps: discovery, sampling, requirements-vision, per-course-extraction, classification. Makes it obvious where the 5 minutes is going.
