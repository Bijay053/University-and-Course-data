Run your verification command, but also extend it to confirm the provenance fields are populated correctly, not just present. Add this query alongside the one you proposed:
sql-- Field population check — confirm provenance fields aren't null/empty
SELECT
    COUNT(*) AS total_backfill_rows,
    COUNT(sfe.snippet::jsonb->>'source_url') AS has_source_url,
    COUNT(sfe.snippet::jsonb->>'source_method') AS has_source_method,
    COUNT(sfe.snippet::jsonb->>'source_conf') AS has_source_conf,
    COUNT(sfe.snippet::jsonb->>'bucket') AS has_bucket,
    COUNT(sfe.snippet::jsonb->>'consensus_count') AS has_consensus_count,
    COUNT(*) FILTER (WHERE (sfe.snippet::jsonb->>'source_conf')::numeric > 0) AS conf_is_nonzero,
    COUNT(*) FILTER (WHERE (sfe.snippet::jsonb->>'consensus_count')::int > 0) AS consensus_is_nonzero
FROM scraped_field_evidence sfe
JOIN scraped_courses sc ON sc.id = sfe.scraped_course_id
JOIN universities u ON u.id = sc.university_id
WHERE u.slug = 'torrens'
  AND sfe.extraction_method ILIKE 'sibling_cache%';
Expected: total_backfill_rows == has_source_url == has_source_method == has_source_conf == has_bucket == has_consensus_count. Any column with a lower count means that field is null on some rows — provenance is incomplete and Prompts 4 and 6 will be unreliable.

SELECT

Expected: recorded_bucket matches the implied bucket from degree_level (e.g. degree_level "Bachelor" → bucket "undergraduate"). If a Bachelor course shows bucket "postgraduate", the bucket assignment is wrong and Prompt 4's source-type gating will fire on the wrong bucket.
