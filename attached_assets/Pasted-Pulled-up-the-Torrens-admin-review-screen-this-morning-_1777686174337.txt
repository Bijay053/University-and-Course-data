**Bug 1 — the cache fix worked, but the underlying problem is bigger than caching**

Looked at Master of Professional Accounting evidence panel. IELTS overall = 6.5, source: `per_course_vision` (note: NOT `per_course_vision_cached`). Evidence image: `how-to-apply/how-to-apply-1-tua.jpg`. That's a generic site-wide "How to apply" marketing image, not anything specific to the Accounting course.

The composite cache key fix prevented contamination between courses (good). But it didn't address that **vision is happily mining IELTS values from generic marketing imagery on every course page**. The "how-to-apply" image presumably contains generic copy like "IELTS 6.5 / PTE 62" because it's the universal application-requirements graphic, and vision treats any number it can OCR as a candidate.

In the Accounting case the value (6.5) happens to match what regex also found on the page, so the row's value is correct. But the evidence chain is junk — and on courses where a generic marketing image happens to show different numbers than the actual course requirement, vision would silently produce a wrong value.

This means last night's audit (`0 promoted vision_cached`) was correct for the question it asked but asked the wrong question. The real audit should also count `per_course_vision` (without `_cached`):

```sql
SELECT 
  sc.extraction_method->>'ielts_overall' AS ielts_source,
  COUNT(*) AS promoted_count
FROM courses c
JOIN scraped_courses sc ON sc.course_id = c.id
WHERE sc.extraction_method->>'ielts_overall' IN ('per_course_vision', 'per_course_vision_cached')
  AND sc.status = 'promoted'
GROUP BY 1;
```

If that returns non-zero, we have an unknown number of live student-facing rows where IELTS came from generic marketing imagery rather than course-specific content.

**The deeper fix** is a relevance check before vision values are accepted: does the image URL path or alt-text relate to the course (course slug match, faculty match, or at least not on a global block-list of generic marketing image paths like `/shared/how-to-apply/`, `/shared/cards/`, `/shared/career-opportunities/`)? If not, don't trust the OCR'd numbers from it.
