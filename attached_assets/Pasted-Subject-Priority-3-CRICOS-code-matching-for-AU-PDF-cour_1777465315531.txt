Subject: Priority 3 — CRICOS code matching for AU PDF↔course pairing
Australian course pages and university-wide PDFs (fee schedules, English requirements docs) both publish CRICOS codes — unique 6-7 digit + letter identifiers registered with the Australian government. Currently we match PDF rows to course pages using fuzzy name overlap, which produces wrong pairings for similarly-named programs (Master of IT - Cyber Security accidentally inheriting Master of IT - AI's fee, etc.).
CRICOS codes are deterministic. When both sides have one, the match is unambiguous and confidence should be near 1.0. When either side is missing the code, fall back to existing fuzzy matching.
This is AU-specific by design. Universities outside Australia won't have CRICOS codes; the matching logic will skip the CRICOS step for them and use fuzzy matching as today. No regression to non-AU scraping.
Estimated effort: half a day.
Component 1: CRICOS code extractor (course pages)

python"""CRICOS code extraction from Australian course pages.

CRICOS = Commonwealth Register of Institutions and Courses for Overseas
Students. Australian government register; unique program identifier per
course version.

Format: 6-7 digits + 1 uppercase letter (e.g., '084932E', '0101389').
"""

import re
from bs4 import BeautifulSoup

# Regex patterns ordered by specificity
CRICOS_LABEL_PATTERNS = [
    # "CRICOS Code: 084932E" / "CRICOS code: 084932E" / "CRICOS#: 084932E"
    re.compile(r"CRICOS\s*(?:Code|code|number|#)?\s*[:\-]?\s*([0-9]{6,7}[A-Z])\b", re.IGNORECASE),
    # "CRICOS 084932E" (bare label + code)
    re.compile(r"\bCRICOS\s+([0-9]{6,7}[A-Z])\b"),
]

# Bare CRICOS code without label — only used as last-resort
CRICOS_BARE_PATTERN = re.compile(r"\b([0-9]{6,7}[A-Z])\b")

# CRICOS-like noise to ignore (postal codes, room numbers, etc.)
CRICOS_NOISE_PATTERNS = [
    re.compile(r"\bROOM\s+\d+[A-Z]"),
    re.compile(r"\bBLOCK\s+\d+[A-Z]"),
]


def extract_cricos_code(html: str | None, text: str | None) -> str | None:
    """Extract CRICOS code from a course page.
    
    Returns the first labeled CRICOS code found, or None if none present.
    Always returns the code in uppercase canonical form.
    """
    if not html and not text:
        return None
    
    # Prefer labeled patterns (most reliable)
    for source in [text, html]:
        if not source:
            continue
        for pattern in CRICOS_LABEL_PATTERNS:
            match = pattern.search(source)
            if match:
                code = match.group(1).upper()
                if not _is_noise(source, match.start()):
                    return code
    
    # Bare-code fallback only on text near a CRICOS keyword
    if text and "cricos" in text.lower():
        # Look for bare codes within 50 chars of any 'cricos' mention
        for cricos_match in re.finditer(r"cricos", text, re.IGNORECASE):
            start = cricos_match.end()
            window = text[start:start + 80]
            bare_match = CRICOS_BARE_PATTERN.search(window)
            if bare_match:
                return bare_match.group(1).upper()
    
    return None


def extract_cricos_code_from_html_structured(html: str) -> str | None:
    """Try to find CRICOS in structured HTML elements (more reliable than regex)."""
    if not html:
        return None
    soup = BeautifulSoup(html, "html.parser")
    
    # Look in definition lists, tables, and key-value structures
    for elem in soup.find_all(["dt", "th", "strong", "label", "span"]):
        if elem.string and "cricos" in elem.string.lower():
            # Sibling or parent might have the code
            sibling = elem.find_next_sibling(["dd", "td", "span"])
            if sibling and sibling.string:
                match = CRICOS_BARE_PATTERN.search(sibling.string)
                if match:
                    return match.group(1).upper()
            # Or it might be in the parent's text
            parent_text = elem.parent.get_text(" ", strip=True) if elem.parent else ""
            for pattern in CRICOS_LABEL_PATTERNS:
                match = pattern.search(parent_text)
                if match:
                    return match.group(1).upper()
    
    return None


def _is_noise(text: str, position: int) -> bool:
    """Check if a CRICOS-like match is actually noise (room number, etc.)."""
    window = text[max(0, position - 40):position + 40]
    return any(p.search(window) for p in CRICOS_NOISE_PATTERNS)

def extract_cricos_from_pdf_row(row_cells: list[str]) -> str | None:
    """Search all cells in a PDF row for a CRICOS-format code.
    
    PDFs have inconsistent column layouts. CRICOS might be in column 1, 3,
    embedded in the program-name cell, or in a separate row above/below.
    """
    for cell in row_cells:
        if not cell or not isinstance(cell, str):
            continue
        match = CRICOS_IN_ROW_PATTERN.search(cell)
        if match:
            return match.group(1).upper()
    return None

Important — adjacent-row CRICOS: some PDFs put the CRICOS code on a row immediately above or below the program name (formatted as a "header" or "subtitle"). Add a second pass that, for any record without cricos_code, checks the immediately preceding and following rows for a bare CRICOS-format value:
pythondef fill_adjacent_row_cricos(records: list[ProgramRecord], 
                              raw_rows: list[list[str]]) -> None:
    """For records missing CRICOS, check adjacent PDF rows."""
    for i, record in enumerate(records):
        if record.cricos_code:
            continue
        # Check row above and below in the original raw_rows
        for offset in [-1, 1]:
            adj_idx = i + offset
            if 0 <= adj_idx < len(raw_rows):
                code = extract_cricos_from_pdf_row(raw_rows[adj_idx])
                if code:
                    record.cricos_code = code
                    break

pythondef match_pdf_record_to_course(
    course: CoursePayload,
    pdf_records: list[ProgramRecord],
) -> tuple[ProgramRecord | None, str]:
    """Match a course to its PDF record. Returns (record, match_method).
    
    Match priority:
    1. CRICOS code equality (deterministic, confidence 0.95)
    2. Fuzzy name match (existing behavior, confidence 0.70)
    
    Returns (None, 'no_match') if neither finds a match.
    """
    # 1. CRICOS-first
    if course.cricos_code:
        cricos_matches = [
            r for r in pdf_records 
            if r.cricos_code and r.cricos_code == course.cricos_code
        ]
        if len(cricos_matches) == 1:
            return cricos_matches[0], "cricos_match"
        elif len(cricos_matches) > 1:
            # Same CRICOS in multiple PDF rows is unusual — could mean
            # the PDF lists per-campus or per-intake variants. Pick the
            # first one but log a warning.
            log.warning(f"Multiple PDF rows match CRICOS {course.cricos_code} "
                       f"for {course.url}: {len(cricos_matches)} matches")
            return cricos_matches[0], "cricos_match_ambiguous"
    
    # 2. Fall back to existing fuzzy match
    fuzzy_match = fuzzy_name_match(course, pdf_records)
    if fuzzy_match:
        return fuzzy_match, "fuzzy_match"
    
    return None, "no_match"

Edge cases worth handling but not blocking

CRICOS revoked/changed: a program's CRICOS code can change between PDF version and current page. The matcher will treat them as non-matching, which is correct — the data really is from different program versions.
Multiple campuses, same CRICOS: rare but happens. The current spec picks the first match and logs a warning. This is acceptable; the logged ambiguity will tell us if it's a real issue.
Domestic-only courses: no CRICOS code (Australian-only programs aren't CRICOS-registered). These will fall through to fuzzy matching, same as today. No special handling needed.

Out of scope for this priority

Generalising to other countries' program codes (NZQA, UCAS, etc.) — separate priority, only worth doing if AU CRICOS matching proves valuable
Building a CRICOS-code-to-course-name lookup table for cross-validation — overengineering for now
Backfilling CRICOS codes on existing scraped_courses rows — the next scheduled scrape will populate them naturally
