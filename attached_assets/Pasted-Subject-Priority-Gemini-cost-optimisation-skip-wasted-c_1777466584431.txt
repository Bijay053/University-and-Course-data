Subject: Priority — Gemini cost optimisation (skip wasted calls + quota protection + cost reporting)
Gemini API spending is currently uncontrolled. Run logs show three categories of waste:

Calling Gemini when regex/CSS extractors already have all the data. UniSQ field summaries consistently show regex extracting fee, IELTS, duration, intake, and study mode at high confidence — then Gemini runs anyway and returns mostly nulls or duplicate values. Pure waste.
No quota protection. When Gemini quota exhausts mid-batch (this has happened multiple times in the last week), every subsequent call fails silently with (likely quota exhausted) and the scrape continues producing degraded data. No circuit breaker, no backoff, no escalation.
No cost visibility. We have per-call costs in run logs but no aggregate reporting. Can't answer "which university costs most to scrape" or "did this week cost more than last week."

This priority addresses all three with a single coordinated change. Estimated 1 day.
Component 1: Skip Gemini when extractors already have the data
The current orchestrator calls Gemini on every course unconditionally. Add a "completeness check" that runs before the Gemini call: if regex/CSS extractors have already populated all the high-priority fields with sufficient confidence, skip Gemini entirely.
python# backend-py/app/services/scraper/gemini_gate.py

# Fields Gemini is most useful for. If all of these are already populated
# with high confidence by other extractors, Gemini adds little.
GEMINI_HIGH_VALUE_FIELDS = {
    "international_fee",
    "ielts_overall",
    "duration_value",
    "intake_months",
    "course_name",
    "study_mode",
    "category",  # Gemini IS good at this — but only if everything else is missing
}

# Fields that Gemini is uniquely good at extracting. Even if the others
# are populated, run Gemini for these.
GEMINI_UNIQUE_FIELDS = {
    "category",        # taxonomy classification
    "sub_category",    # taxonomy classification
    "description",     # narrative summarization (sometimes)
}

CONFIDENCE_THRESHOLD = 0.70  # only count fields that pass this bar


def should_skip_gemini_primary(payload: CoursePayload, 
                                evidence_so_far: dict) -> tuple[bool, str]:
    """Decide whether to call Gemini's primary text extraction for a course.
    
    Returns (should_skip, reason). Reasons:
    - 'all_high_value_fields_populated': skip — regex/CSS got everything
    - 'classification_only': run, but with classification-only prompt (cheaper)
    - 'full_extraction_needed': run with full prompt (most expensive)
    """
    populated_high_value = {
        field for field in GEMINI_HIGH_VALUE_FIELDS
        if (payload.get(field) is not None
            and evidence_so_far.get(field, {}).get("confidence", 0) >= CONFIDENCE_THRESHOLD)
    }
    
    coverage = len(populated_high_value) / len(GEMINI_HIGH_VALUE_FIELDS)
    
    # If 90%+ of high-value fields are populated by other extractors,
    # only run Gemini if classification (category/sub_category) is missing.
    if coverage >= 0.90:
        if not payload.get("category") or not payload.get("sub_category"):
            return False, "classification_only"  # run with cheaper prompt
        return True, "all_high_value_fields_populated"
    
    return False, "full_extraction_needed"


def build_classification_only_prompt(course_name: str, page_text: str) -> str:
    """Cheaper prompt that only asks for category and sub_category.
    
    Used when other fields are already populated. Saves both input tokens
    (smaller prompt) and output tokens (smaller schema).
    """
    # Truncate page text aggressively — we only need enough to classify
    page_snippet = page_text[:1500] if page_text else ""
    
    return f"""Given this course information, return ONLY the category and sub_category.

Course name: {course_name}
Page excerpt: {page_snippet}

Respond with JSON containing only:
{{
  "category": "...",       // top-level field e.g. "Business & Management"
  "sub_category": "..."   // specific area e.g. "Marketing"
}}
"""

Component 2: Quota wrapper with circuit breaker
Wrap all Gemini calls in a quota-aware client that tracks failures and short-circuits when quota is clearly exhausted, instead of letting every subsequent call hit the API and fail.
python# backend-py/app/services/scraper/gemini_client.py

from datetime import datetime, timedelta
from collections import deque
import asyncio

class GeminiQuotaTracker:
    """Tracks recent failures and trips a circuit breaker when quota
    is likely exhausted. Persists state across requests in the same process.
    """
    
    def __init__(self):
        self.recent_failures = deque(maxlen=20)
        self.circuit_open_until: datetime | None = None
        self.failure_threshold = 5         # 5 failures within window
        self.window_seconds = 60            # 60-second window
        self.cool_down_seconds = 300        # circuit stays open for 5 min after trip
    
    def record_failure(self, error_code: int | None, error_message: str):
        """Record a Gemini API failure. May trip the circuit breaker."""
        now = datetime.now()
        is_quota_error = self._looks_like_quota(error_code, error_message)
        
        if is_quota_error:
            self.recent_failures.append(now)
            recent_window = [t for t in self.recent_failures 
                            if (now - t).total_seconds() < self.window_seconds]
            
            if len(recent_window) >= self.failure_threshold:
                self.circuit_open_until = now + timedelta(seconds=self.cool_down_seconds)
                log.warning(f"[GEMINI CIRCUIT OPEN] {len(recent_window)} quota errors "
                           f"in {self.window_seconds}s — pausing calls until "
                           f"{self.circuit_open_until.isoformat()}")
    
    def is_circuit_open(self) -> bool:
        if not self.circuit_open_until:
            return False
        if datetime.now() >= self.circuit_open_until:
            log.info("[GEMINI CIRCUIT CLOSED] cool-down complete")
            self.circuit_open_until = None
            self.recent_failures.clear()
            return False
        return True
    
    def time_until_circuit_close(self) -> float:
        if not self.circuit_open_until:
            return 0
        return max(0, (self.circuit_open_until - datetime.now()).total_seconds())
    
    def _looks_like_quota(self, error_code: int | None, message: str) -> bool:
        if error_code in (429, 503):
            return True
        if message and any(kw in message.lower() for kw in [
            "quota", "rate limit", "exhausted", "exceeded", "resource_exhausted"
        ]):
            return True
        return False


# Singleton — shared across the process
_quota_tracker = GeminiQuotaTracker()


async def call_gemini_safe(*args, **kwargs):
    """Wrapper around Gemini API client. Honors circuit breaker.
    
    Returns None if circuit is open or call fails (existing callers should
    already handle None responses gracefully).
    """
    if _quota_tracker.is_circuit_open():
        wait_time = _quota_tracker.time_until_circuit_close()
        log.info(f"[GEMINI SKIP-CIRCUIT] circuit open for {wait_time:.0f}s more")
        return None
    
    try:
        response = await actual_gemini_call(*args, **kwargs)
        return response
    except QuotaExceededError as e:
        _quota_tracker.record_failure(429, str(e))
        return None
    except Exception as e:
        _quota_tracker.record_failure(getattr(e, "code", None), str(e))
        # Don't raise — caller already handles None
        return None
When circuit is open, the orchestrator continues processing courses but skips Gemini entirely. Other extractors (regex, CSS, vision) keep working. Stagings produced during circuit-open periods get tagged with a degraded-quality flag for later re-scrape.

class JobCostMonitor:
    """Tracks Gemini spend for a scrape job. Aborts if budget is exceeded."""

        self.scrape_run_id = scrape_run_id
        self.university_code = university_code
        self.budget_usd = budget_usd
        self.spent_usd = 0.0
        self.aborted = False

        self.spent_usd += cost_usd

                     f"exceeded budget: ${self.spent_usd:.4f} >= ${self.budget_usd:.2f}")
            # Emit critical alert
            asyncio.create_task(self._emit_alert())
    
    def can_continue(self) -> bool:
        return not self.aborted

sql-- Per-university cost over time
CREATE VIEW v_gemini_cost_by_university AS
SELECT 
    u.code,
    u.name,
    DATE_TRUNC('day', g.created_at) AS day,
    COUNT(*) AS total_calls,
    COUNT(*) FILTER (WHERE g.success = true) AS successful_calls,
    SUM(g.cost_usd) AS daily_cost_usd,
    AVG(g.cost_usd) AS avg_cost_per_call,
    AVG(g.input_tokens) AS avg_input_tokens
FROM gemini_call_log g
JOIN universities u ON u.id = g.university_id
GROUP BY u.code, u.name, DATE_TRUNC('day', g.created_at);

-- Per-call-type cost breakdown
CREATE VIEW v_gemini_cost_by_call_type AS
SELECT 
    call_type,
    DATE_TRUNC('day', created_at) AS day,
    COUNT(*) AS calls,
    SUM(cost_usd) AS total_cost_usd,
    AVG(input_tokens) AS avg_input_tokens,
    AVG(output_tokens) AS avg_output_tokens
FROM gemini_call_log
WHERE success = true
GROUP BY call_type, DATE_TRUNC('day', created_at);

-- Top 10 most expensive universities (last 30 days)
CREATE VIEW v_gemini_top_spenders_30d AS
SELECT 
    u.code,
    u.name,
    COUNT(*) AS total_calls,
    SUM(g.cost_usd) AS total_cost_usd,
    SUM(g.cost_usd) / COUNT(*) AS avg_cost_per_call,
    SUM(g.cost_usd) / NULLIF(COUNT(DISTINCT g.scrape_run_id), 0) AS avg_cost_per_scrape
FROM gemini_call_log g
JOIN universities u ON u.id = g.university_id
WHERE g.created_at > NOW() - INTERVAL '30 days'
GROUP BY u.code, u.name
ORDER BY total_cost_usd DESC
LIMIT 10;

-- Skip rate — what % of courses had Gemini skipped?
CREATE VIEW v_gemini_skip_efficiency AS
SELECT 
    DATE_TRUNC('day', sj.completed_at) AS day,
    COUNT(DISTINCT sc.id) AS total_courses,
    COUNT(DISTINCT g.course_url) AS courses_called_gemini,
    COUNT(DISTINCT sc.id) - COUNT(DISTINCT g.course_url) AS courses_skipped_gemini,
    (COUNT(DISTINCT sc.id) - COUNT(DISTINCT g.course_url))::float 
        / NULLIF(COUNT(DISTINCT sc.id), 0) AS skip_rate
FROM scrape_runtime_jobs sj
JOIN scraped_courses sc ON sc.scrape_run_id = sj.id
LEFT JOIN gemini_call_log g ON g.scrape_run_id = sj.id 
    AND g.call_type IN ('primary_full', 'classification_only')
WHERE sj.status = 'completed'
GROUP BY DATE_TRUNC('day', sj.completed_at);

Risk considerations

Skip logic could miss edge cases: a university where regex extracts wrong values at high confidence would now never get Gemini's correction. Mitigation: the confidence threshold (0.70) is relatively high, and fall-through still happens for mis-classified courses. Monitor skip rate vs data quality alerts (Priority 5) for the first week post-deploy.
Circuit breaker too aggressive: if Gemini has transient errors that aren't quota-related, the circuit could trip unnecessarily. Mitigation: only trips on errors matching quota patterns (429, 503, "exhausted", "rate limit"). Other errors don't count.
Cost ceiling might block legitimate large universities: budgets are set per-university with LARGE_UNI_BUDGETS. If a uni hits its budget legitimately, increase the budget for that uni rather than letting it abort silently. Alerts will surface this.
