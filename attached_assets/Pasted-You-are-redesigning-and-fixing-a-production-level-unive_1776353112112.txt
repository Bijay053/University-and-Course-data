Main goal:
Build a smart, low cost, highly accurate university data scraping engine that can discover, extract, validate, and stage complete course data for a university portal, course search engine, and agent application portal.

==================================================
CORE PRINCIPLE
==================================================

Do not scrape blindly.

Before extracting any data, do a research phase:
1. Identify whether the given URL is home page, course search page, course list page, sitemap, faculty page, scholarship page, fee page, entry requirement page, or a single course page
2. Discover the best course listing page automatically if the user only gives the home page
3. Compare multiple candidate pages before deciding where to scrape
4. Estimate expected course count before starting large extraction
5. Confirm which pages are real course pages
6. Reject navigation pages, blog pages, support pages, accommodation pages, visa pages, news pages, event pages, generic marketing pages, and student life pages
7. Only continue if confidence is high

Build a research first architecture, not just a fetch and hope system.

1. ADVANCED COURSE DISCOVERY LOGIC
==================================================

Goal:
Identify real course pages only and avoid junk pages.

Research logic:
1. Search candidate sources in this order:
   a. dedicated course listing page
   b. sitemap or course sitemap
   c. faculty or school pages containing course links
   d. internal search results pages
   e. structured API or JSON data if site exposes it
2. Score each candidate page using content signals
3. Compare candidates and pick the best source of truth

A page is likely a real course page if it contains several of these:
- course title
- degree keywords such as Bachelor, Master, Diploma, Graduate Certificate, Graduate Diploma, PhD, Doctor of Philosophy, Associate Degree, Certificate
- duration
- campus or location
- study mode
- intake information
- entry requirements
- tuition fee
- English language requirements
- how to apply
- course structure
- career outcomes

Reject a page if it looks like:
- accommodation
- student life
- scholarships overview
- events
- news
- visas
- support
- contact us
- why choose us
- blog
- pathway overview without specific course data
- landing page with no real course data

Listing page logic:
- detect total course count if shown
- detect pagination or load more
- follow all pages only after validation
- deduplicate URLs
- normalize URLs
- remove domestic and international parameter duplicates unless needed for comparison

Do not assume every page under /courses/ is a course.
Validate by content before accepting.

2. ADVANCED FEE DISCOVERY LOGIC
==================================================

Goal:
Extract only the correct international tuition fee for the course.

Search logic:
For each course, search for fee from:
1. main course page
2. fee section on same page
3. linked tuition fee page
4. international fees page
5. fee table page
6. downloadable PDF if relevant
7. faculty fee page
8. handbook or course guide if official

Important rules:
- always prioritize international fee
- always exclude domestic fee from the main extracted value
- if domestic and international are both present, choose international only
- if multiple fee values are present, link them to the right course and location where possible
- if fee varies by year, campus, or intake, store the current valid value and also keep source evidence
- if fee is listed by subject or credit points only, calculate only if the formula is clearly reliable, otherwise mark manual review
- if a fee page is generic and not course specific, link only if degree or faculty mapping is confident

Recognize fee phrases such as:
- Tuition fee
- International tuition fee
- Annual tuition fee
- Estimated annual fee
- Total tuition fee
- Full course fee
- Fee per semester
- Fee per trimester
- Fee per term
- Indicative annual fee

Exclude phrases such as:
- domestic fee
- local student fee
- commonwealth supported
- government subsidized
- FEE HELP
- student services fee unless separately stored

Confidence rules:
- high confidence if the course page directly states international fee
- medium confidence if fee is found on official linked fee page and clearly matches course or level
- low confidence if inferred from generic faculty table

==================================================

Goal:

Search sources:
1. main course page

Then save as:
- IELTS Overall = 6.0
- IELTS Listening = 5.5
- IELTS Reading = 5.5
- IELTS Writing = 5.5
- IELTS Speaking = 5.5

Apply the same rule for PTE and TOEFL when wording follows similar pattern:
- PTE Overall 50 with no communicative skill below 42
Then save:
- PTE Overall = 50
- PTE Listening = 42
- PTE Reading = 42
- PTE Writing = 42
- PTE Speaking = 42

- TOEFL Overall 60 with no band less than 18
Then save:
- TOEFL Overall = 60
- TOEFL Listening = 18
- TOEFL Reading = 18
- TOEFL Writing = 18
- TOEFL Speaking = 18

Support these wording patterns:
- no band less than
- no skill below
- minimum of X in each component
- minimum score of X in writing and Y in other bands
- overall X with subscores of Y
- total score X with section minimum Y

If test scores differ by program type or specialization:
- map to the specific course where possible
- if multiple score sets exist and cannot be confidently mapped, mark for manual review

If only overall is listed and no subscore is mentioned:
- save overall only
- leave L R W S null
- do not guess

If multiple English tests are accepted:
- extract all
