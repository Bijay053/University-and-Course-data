3. The [VISION SKIP] log line is the single most important diagnostic for this PR.
Your expected output:
[VISION SKIP] trust_vision_ocr=false for this uni — skipping vision OCR pass on https://www.acap.edu.au/courses/...
Two failure modes to watch for:
(a) Line never appears in the log — gate isn't firing. Helper might be returning True when it should be False. Check that set_uni_config() is actually loading acap_41.yaml (not just the schema default) when ACAP scrape runs. Quick verify: add a one-time debug log in the helper showing the resolved value from contextvar, run scrape once, check value is False, remove debug log. (b) Line appears but [VISION] N candidate images ALSO appears — gate is wrapping the wrong block. The log should show one or the other, never both for the same course. If both fire, the gate is logging the skip but the vision pass runs anyway downstream.
Both failure modes will show up cleanly in the ACAP scrape log right after pull. Read the log before running the sweep — if the gate isn't firing correctly, the sweep will show diff anyway and you'll waste cycles debugging at the wrong layer.
