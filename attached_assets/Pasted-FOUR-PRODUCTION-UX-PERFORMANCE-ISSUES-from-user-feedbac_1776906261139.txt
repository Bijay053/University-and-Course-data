ISSUE 2: Scraping page doesn't show in-progress scrapes when opened in a new browser tab
User opens /scraping page in Browser A, starts a VIT scrape, it's running. Opens /scraping in Browser B at the same time — shows empty state, no indicator that a scrape is running.
Root cause: The page loads in-progress state from local React state, not from the server. Job info lives only in the browser that started it.
Fix — Load active jobs from server on page mount:
Create endpoint that returns any currently-running scrape:
typescriptrouter.get("/scrape/active", async (_req, res) => {
  const running = await db.query.scrapeRuntimeJobsTable.findMany({
    where: (j, { inArray }) => inArray(j.status, ['running', 'queued']),
    orderBy: (j, { desc }) => desc(j.startedAt),
    limit: 5,
  });
  res.json({ activeJobs: running });
});
On the Scraping page, useEffect on mount:
typescriptuseEffect(() => {
  fetch('/api/scrape/active')
    .then(r => r.json())
    .then(data => {
      if (data.activeJobs?.length > 0) {
        const job = data.activeJobs[0];
        setCurrentJobId(job.runtimeJobId);
        setUniversityName(job.universityName);
        setIsRunning(true);
        // Then start polling the status endpoint like normal
      }
    });
}, []);
This way any browser tab on the Scraping page picks up whatever is currently running.
